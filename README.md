## Quick Start & Setup

To get a local copy of this project up and running, follow these steps:

### 1. Prerequisites

Ensure you have [Git](https://git-scm.com/) installed. No local server (like Node.js) is required as the project utilizes CDN-based assets.

### 2. Installation

```bash
# Clone the repository
git clone https://github.com/Kwanusu/corporate.git

# Navigate into the project directory
cd corporate

```

### 3. Deployment & Live View

* **Local:** Open `index.html` in any modern browser or use the VS Code **Live Server** extension.
* **Production:** This project is ready for **GitHub Pages**. Push to the `main` branch and enable "Pages" in your repository settings.

## Design Philosophy & Choices

The architecture follows a **Stone & Amber** design language (`stone-950` backgrounds with `amber-500` accents) to evoke prestige and security.

* **HUD-Style Aesthetics:** We utilize `backdrop-blur` and `opacity` gradients to create a "Heads-Up Display" feel, making the UI feel like an advanced technical instrument.
* **The Spec-Reveal Architecture:** Service cards use a dual-layer `translate-y` transition. This prevents "information overload" by hiding technical matrices (latency, encryption, etc.) until the user actively engages.
* **Micro-Glows:** Absolute-positioned divs with `blur` utilities create a "halo" effect, separating content without the need for harsh, distracting borders.

## Contribution Guide

To maintain the **Innovet Tech** standard, please adhere to these protocols:

1. **Code Style:** Use Tailwind utility classes exclusively. Avoid custom CSS.
2. **Naming:** Use lowercase and hyphens for all assets (e.g., `cloud-visual.jpg`). GitHub Pages is case-sensitive.
3. **Workflow:** Fork the repo, create a `feature/` branch, and ensure mobile responsiveness before submitting a Pull Request.
